# 실습 기록

- **환경:** Rocky Linux 10, Wi-Fi 인터페이스(`wlp0s20f3`), DHCP(KT 공유기)
- **서버 IP:** `172.30.1.100/24`
- **게이트웨이:** `172.30.1.254`

---

## Part 1 — 네트워크 상태 확인

### 1-1. L2 인터페이스 확인

```bash
jangwoojung@localhost:~$ ip -br link show
lo               UNKNOWN        00:00:00:00:00:00 <LOOPBACK,UP,LOWER_UP>
wlp0s20f3        UP             fc:44:82:bb:7f:f6 <BROADCAST,MULTICAST,UP,LOWER_UP>
# wlp0s20f3: wl(Wireless LAN) + p0s20f3(PCI 버스 위치)
# UP + LOWER_UP = 인터페이스 활성화 + 물리적 연결 정상
```

```bash
jangwoojung@localhost:~$ ip -s link show wlp0s20f3
    RX:  bytes packets errors dropped  missed   mcast
     381528077  363708      0     817       0       0
    TX:  bytes packets errors dropped carrier collsns
      56553535  124516      0     128       0       0
# RX dropped: 817, TX dropped: 128 — Wi-Fi 환경에서 소량 드롭은 정상 범위
```

### 1-2. L3 IP · 라우팅 · DNS 확인

```bash
jangwoojung@localhost:~$ ip addr show wlp0s20f3
    inet 172.30.1.100/24 brd 172.30.1.255 scope global dynamic noprefixroute wlp0s20f3
       valid_lft 3123sec preferred_lft 3123sec
# dynamic = DHCP로 할당받은 IP
# valid_lft 3123sec = DHCP 임대 시간 잔여량
```

```bash
jangwoojung@localhost:~$ ip route show default
default via 172.30.1.254 dev wlp0s20f3 proto dhcp src 172.30.1.100 metric 600
# via 172.30.1.254 = 기본 게이트웨이 (KT 공유기)
# proto dhcp = DHCP가 자동 설정한 경로
```

```bash
jangwoojung@localhost:~$ cat /etc/resolv.conf
# Generated by NetworkManager
nameserver 168.126.63.1
nameserver 168.126.63.2
# KT ISP의 DNS 서버. NetworkManager가 DHCP 응답을 받아 자동 기록
```

### 1-3. ARP 테이블 확인

```bash
jangwoojung@localhost:~$ ip neigh show
172.30.1.254 dev wlp0s20f3 lladdr 0c:96:cd:a3:b8:ab REACHABLE
# 게이트웨이의 MAC 주소가 REACHABLE — L2 통신 정상
```

### 1-4. 소켓 · 포트 확인

```bash
jangwoojung@localhost:~$ ss -tulnp
Netid  State   Recv-Q  Send-Q  Local Address:Port   Peer Address:Port  Process
tcp    LISTEN  0       128           0.0.0.0:22            0.0.0.0:*              # sshd
tcp    LISTEN  0       4096        127.0.0.1:631           0.0.0.0:*              # CUPS (로컬 전용)
tcp    LISTEN  0       4096              [::]:22              [::]:*              # sshd (IPv6)
tcp    LISTEN  0       4096                *:9090               *:*              # cockpit 웹 콘솔
```

### 1-5. 단계별 ping (연결성 확인)

```bash
jangwoojung@localhost:~$ ping -c 2 127.0.0.1
2 packets transmitted, 2 received, 0% packet loss
rtt min/avg/max/mdev = 0.072/0.072/0.073/0.000 ms
# ✓ loopback — 커널 네트워크 스택 정상

jangwoojung@localhost:~$ ping -c 2 172.30.1.100
2 packets transmitted, 2 received, 0% packet loss
rtt min/avg/max/mdev = 0.081/0.090/0.100/0.009 ms
# ✓ 자기 자신 IP — NIC에 IP 할당 정상

jangwoojung@localhost:~$ ping -c 2 8.8.8.8
2 packets transmitted, 2 received, 0% packet loss
rtt min/avg/max/mdev = 31.897/32.700/33.503/0.803 ms
# ✓ 외부 IP — 게이트웨이 통한 인터넷 라우팅 정상

jangwoojung@localhost:~$ ping -c 2 google.com
PING google.com (172.217.213.139) 56(84) bytes of data.
2 packets transmitted, 2 received, 0% packet loss
rtt min/avg/max/mdev = 29.865/31.228/32.591/1.363 ms
# ✓ 도메인 — DNS 해석 정상
```

4단계 모두 성공 → L2/L3/라우팅/DNS 전부 정상. 외부 RTT ~32ms는 Wi-Fi + ISP 경유 지연.

> **실습 중 실수:** `ping -c 8.8.8.8`처럼 `-c` 뒤에 횟수를 빠뜨리면 에러가 난다. 올바른 사용법은 `ping -c 2 8.8.8.8` (횟수 + 대상).

### 1-8. connection profile 파일 확인

```bash
jangwoojung@localhost:~$ cat /etc/NetworkManager/system-connections/KT_GiGA_5G_B8AA.nmconnection
cat: /etc/NetworkManager/system-connections/KT_GiGA_5G_B8AA.nmconnection: Permission denied

jangwoojung@localhost:~$ sudo cat /etc/NetworkManager/system-connections/KT_GiGA_5G_B8AA.nmconnection
[connection]
id=KT_GiGA_5G_B8AA
uuid=12be06dc-aed7-4571-a8ee-02b0c1cfa961
type=wifi
interface-name=wlp0s20f3

[wifi]
mode=infrastructure
ssid=KT_GiGA_5G_B8AA

[wifi-security]
auth-alg=open
key-mgmt=wpa-psk
psk=bzc87fg747              # Wi-Fi 비밀번호가 평문으로 저장됨

[ipv4]
method=auto                 # DHCP 모드

[ipv6]
addr-gen-mode=default
method=auto

[proxy]
```

`sudo` 없이 읽으면 `Permission denied` — 파일 퍼미션이 600(root만 읽기)이기 때문이다. `[wifi-security]`에 PSK(비밀번호)가 평문으로 저장되므로, 이 퍼미션이 반드시 필요하다.

## Part 2 — firewalld 방화벽 실습

### 2-1. httpd 설치 및 시작

```bash
jangwoojung@localhost:~$ sudo dnf install httpd -y
Installed:
  httpd-2.4.63-4.el10_1.3.x86_64  (+ 의존성 10개)
Complete!

jangwoojung@localhost:~$ systemctl start httpd
jangwoojung@localhost:~$ systemctl enable httpd
Created symlink '/etc/systemd/system/multi-user.target.wants/httpd.service'
  → '/usr/lib/systemd/system/httpd.service'.
```

### 2-2. 방화벽 현재 상태 확인

```bash
jangwoojung@localhost:~$ firewall-cmd --get-default-zone
public

jangwoojung@localhost:~$ firewall-cmd --list-all
public (default, active)
  target: default                          # 허용 목록에 없으면 REJECT
  interfaces: enp0s20f0u3 wlp0s20f3
  services: cockpit dhcpv6-client ssh      # http 없음 = 외부에서 80포트 차단 중
  ports:
  rich rules:
```

### 2-3. 방화벽 열기 전 접근 테스트 (차단 체감)

```bash
jangwoojung@localhost:~$ ss -tlnp | grep :80
LISTEN 0      511                *:80              *:*
# httpd가 80 포트에서 LISTEN 중 — 프로세스는 살아있다

jangwoojung@localhost:~$ curl http://localhost/
<!doctype html> ... HTTP Server Test Page powered by: Rocky Linux ...
# 로컬 접근 성공 — loopback은 netfilter INPUT 체인을 거치지 않음
```

```bash
# 다른 머신(Mac)에서 접근 시도
Jangwoo 🚀 ~ $ curl http://172.30.1.100/
curl: (7) Failed to connect to 172.30.1.100 port 80 after 1 ms: Couldn't connect to server
# 외부 접근 차단 — 프로세스는 LISTEN 중인데 방화벽이 인바운드를 막고 있다
```

프로세스가 살아있어도 방화벽에서 서비스를 허용하지 않으면 외부에서 접근할 수 없다는 것을 직접 확인했다.

### 2-4. 서비스 허용 + 검증

```bash
jangwoojung@localhost:~$ firewall-cmd --zone=public --add-service=http --add-service=https --permanent
success
jangwoojung@localhost:~$ firewall-cmd --reload
success
```

```bash
jangwoojung@localhost:~$ echo "Hello Jangwoo server" | sudo tee /var/www/html/index.html
Hello Jangwoo server
```

> **실습 중 실수:** `sudo echo "..." > /var/www/html/index.html`은 Permission denied가 발생한다. 리다이렉션(`>`)은 현재 셸(일반 사용자) 권한으로 실행되기 때문이다. `sudo tee` 또는 `sudo bash -c 'echo "..." > ...'`를 사용해야 한다.

```bash
jangwoojung@localhost:~$ curl http://172.30.1.100/
Hello Jangwoo server
# 외부 IP로 접근 성공 — 방화벽에 http 서비스 추가 후 정상 동작
```

### 2-5. 비표준 포트 허용

```bash
jangwoojung@localhost:~$ firewall-cmd --zone=public --add-port=8080/tcp --permanent
success
jangwoojung@localhost:~$ firewall-cmd --reload
success
jangwoojung@localhost:~$ firewall-cmd --list-ports
8080/tcp
```

### 2-6. 런타임 vs 영구 비교

```bash
jangwoojung@localhost:~$ firewall-cmd --zone=public --add-port=9090/tcp   # --permanent 없이
success

jangwoojung@localhost:~$ firewall-cmd --list-ports
8080/tcp 9090/tcp                    # 런타임에는 둘 다 존재

jangwoojung@localhost:~$ firewall-cmd --reload
success

jangwoojung@localhost:~$ firewall-cmd --list-ports
8080/tcp                             # 9090 사라짐 — 런타임 전용이었으므로
```

`--permanent` 없이 추가한 규칙은 `--reload` 또는 재부팅 시 소멸한다.

### 2-7. Rich Rule (특정 IP 허용)

```bash
jangwoojung@localhost:~$ firewall-cmd --zone=public \
    --add-rich-rule='rule family="ipv4" source address="10.0.0.50" service name="ssh" accept' \
    --permanent
success
jangwoojung@localhost:~$ firewall-cmd --reload
success

jangwoojung@localhost:~$ firewall-cmd --zone=public --list-rich-rules
rule family="ipv4" source address="10.0.0.50" service name="ssh" accept
```

```bash
# 정리
jangwoojung@localhost:~$ firewall-cmd --zone=public \
    --remove-rich-rule='rule family="ipv4" source address="10.0.0.50" service name="ssh" accept' \
    --permanent
success
jangwoojung@localhost:~$ firewall-cmd --reload
success
```

### 2-8. Rich Rule 로깅

```bash
jangwoojung@localhost:~$ firewall-cmd --zone=public \
    --add-rich-rule='rule family="ipv4" service name="http" log prefix="HTTP_ACCESS: " level="info" accept' \
    --permanent
success
jangwoojung@localhost:~$ firewall-cmd --reload
success

jangwoojung@localhost:~$ curl http://localhost/
Hello Jangwoo server

jangwoojung@localhost:~$ journalctl -k | grep "HTTP_ACCESS"
jangwoojung@localhost:~$
# 결과 없음
```

`curl http://localhost/`는 loopback(127.0.0.1)을 통해 접근하므로 netfilter INPUT 체인을 거치지 않는다. Rich Rule은 netfilter를 통과하는 패킷에만 적용되므로, 로깅 테스트는 외부 IP(`curl http://172.30.1.100/`)로 해야 했다.
