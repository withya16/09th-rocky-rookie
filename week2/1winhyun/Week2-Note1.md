# DNF와 RPM: 패키지 관리 도구의 이해

## 1. RPM (Red Hat Package Manager)

**RPM**은 RHEL, CentOS, Fedora 등 Red Hat 계열 리눅스 시스템에서 소프트웨어를 설치하고 관리하는 **저수준(Low-level) 패키지 형식 및 실행 도구**입니다.

- **기능**: 확장자가 `.rpm`인 개별 소프트웨어 패키지 아카이브를 시스템에 설치, 조회, 업데이트, 제거하며, 패키지의 무결성을 검증합니다.
- **한계 (의존성 문제)**: RPM의 가장 큰 특징이자 한계점은 패키지 간의 복잡한 **의존성(Dependencies)을 자동으로 해결하지 못한다**는 것입니다. 특정 패키지 설치 시 다른 라이브러리가 필요하다면, RPM은 해당 라이브러리가 없다는 오류만 출력하고 설치를 즉시 중단합니다.

---

## 2. DNF (Dandified YUM)

**DNF**는 RPM의 한계인 **'의존성 지옥'**을 해결하기 위해 등장한 **고수준(High-level) 패키지 관리 도구**입니다. 기존에 사용되던 YUM(Yellowdog Updater Modified)의 단점을 개선한 최신 후속 버전입니다.

- **기능**: 네트워크로 연결된 소프트웨어 리포지토리(저장소)에서 패키지를 다운로드하고 관리합니다.
- **의존성 자동 해결**: 사용자가 패키지를 설치할 때, DNF는 `libsolv`라는 라이브러리를 사용하여 필요한 모든 의존 관계를 자동으로 계산하고 관련 패키지들을 함께 설치해 줍니다.

---

## 3. DNF와 RPM의 핵심 차이 및 상호 관계

두 도구의 개념을 명확히 구분하기 위한 핵심 차이와 상호 관계는 다음과 같습니다.

### 3-1. 계층 및 상호 보완적 역할

DNF는 RPM을 **대체하는 것이 아니라** RPM을 기반으로 그 위에서 동작합니다.

| 계층 | 도구 | 역할 |
|------|------|------|
| **논리적 관리 계층** | DNF | 네트워크에서 패키지를 찾고 의존성을 계산 |
| **물리적 실행 계층** | RPM | 시스템 파일 시스템에 파일을 배치하고 데이터베이스(RPMDB)에 정보를 기록 |

### 3-2. 의존성(Dependency) 해결 능력

- **RPM**: 단일 패키지만 다루기 때문에 의존성을 **수동으로** 맞춰주어야 합니다.
- **DNF**: 소프트웨어 저장소의 메타데이터를 분석하여 패키지 설치에 필요한 선행 프로그램이나 라이브러리를 **알아서** 다운로드하여 설치합니다.

### 3-3. 실무에서의 사용 목적

- **DNF 사용**: 일반 사용자나 시스템 관리자가 소프트웨어를 설치, 업데이트, 삭제할 때는 대부분 DNF를 사용하는 것이 권장됩니다.
- **RPM 사용**: RPM 명령어는 패키지의 세부 정보를 직접 뜯어보거나, 저수준의 시스템 트러블슈팅을 할 때 주로 사용됩니다.

> **참고**: 인터넷에서 직접 다운로드한 개별 `.rpm` 파일을 설치할 때도 `rpm -i` 명령보다는 `dnf install 파일명.rpm`을 사용하는 것이 유리합니다. DNF를 사용하면 로컬 패키지가 요구하는 추가 의존성까지 리포지토리에서 알아서 설치해 주기 때문입니다.

---

## 4. 패키지 관리 도구의 진화

리눅스 패키지 관리 아키텍처는 성능과 효율성을 높이는 방향으로 계속 진화하고 있습니다.

```
RPM → YUM → DNF (DNF4) → DNF5
```

| 도구 | 설명 |
|------|------|
| **RPM** | 기본 패키지 설치 엔진. (의존성 해결 불가) |
| **YUM** | RHEL의 구형 관리 도구로, 의존성 자동 해결을 도입했으나 Python 기반으로 인해 메모리 사용량이 많고 속도가 다소 느렸습니다. |
| **DNF (DNF4)** | YUM의 차세대 버전. `libsolv`를 도입해 의존성 계산 속도를 크게 높이고 성능을 개선했습니다. |
| **DNF5** | 최신 기술(Fedora 41 및 RHEL 10에서 기본 채택)로, Python 기반이었던 프론트엔드와 백엔드를 모두 **C++로 완전히 재작성**한 차세대 도구입니다. Python 인터프리터를 구동할 필요가 없어 초기 가동 속도가 즉각적이며, 메모리 점유율을 대폭 낮춘 것이 특징입니다. |
