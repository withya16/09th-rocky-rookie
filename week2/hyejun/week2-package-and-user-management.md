## 1. DNF 및 RPM 개념 및 차이

Redhat, RHEL, Fedora, CentOS 등 운영체제는 Redhat 계열의 리눅스이다. rpm, yum, dnf는 모두 Redhat사에서 패키지를 좀 더 쉽게 설치하고 관리하기 위해 만든 패키지 관리 프로그램!

- **RPM (Red Hat Package Manager):** 확장자명: *.rpm 이며 이런 파일들을 패키지로 부르기도 한다. 로컬 상에 패키지가 있어야 설치가능. 패키지를 다운로드 받는 방법은 크게 CD와 같은 물리적인 디스크로 내려받거나, 인터넷상에 직접 패키지를 다운로드받을수도 있다. 의존성 문제 발생 시 수동 해결 필요.
    - 장점: 복잡한 컴파일 과정 없이 간단하게 패키지 설치 가능
    - 단점: 사용자에게 맞는 다양한 옵션 설정불가. 패키지를 따로 다운받아야 함.

![image.png](image.png)

- Y**UM** : RPM에서 더 발전된 패키지 관리 명령어. 인터넷이 된다는 가정 하에 우리가 원하는 파일, 프로그램들을 자동으로 설치받을 수 있음. RPM에서 패키지를 갖고 있어야 설치가 가능했던 부분을 YUM에서는 스스로 웹에서 다운받아와 설치 → 사용자 접근 편리
    - 장점: RPM의 패키지 의존성 문제를 완전히 해결, 웹을 통해 필요한 파일을 저장소에서 다운로드하여 설치 가능
- **DNF (Dandified YUM):** YUM의 개선 버전. 느린 속도, 과다한 메모리 사용, 의존성 결정이 느린 문제들을 개선. RHEL/CentOS 8에 정식 도입. 기존 사용자들을 위한 yum 유틸리티 제공(dnf 백엔드로 사용) yum과 명령어, 옵션 등과 같은 조작법 거의 비슷.
    - **DNF vs RPM 의 핵심 차이**
        - 즉, 경쟁관계가 아니라 계층적인 보완 관계
        
        | **구분** | **RPM (Red Hat Package Manager)** | **DNF (Dandified YUM)** |
        | --- | --- | --- |
        | **역할** | **저수준(Low-level)** 패키지 도구 | **고수준(High-level)** 패키지 관리자 |
        | **작업 단위** | 개별 `.rpm` 파일 | 리포지토리(저장소) 전체 |
        | **의존성 해결** | **수동** (의존성 오류 시 사용자 직접 설치) | **자동** (의존성 자동 추적 및 설치) |
        | **데이터 소스** | 로컬 시스템의 패키지 파일 | 온라인/오프라인 리포지토리 및 메타데이터 |
        | **관계** | 패키지 포맷이자 기초 엔진 | RPM을 활용하는 상위 프런트엔드 |

## 2. 패키지 관리 (DNF)

- 설치 (`install`): 패키지 및 의존성 라이브러리 자동 설치
- 업데이트 (`update`): 시스템 및 특정 패키지 최신화
- 삭제 (`remove`): 패키지 제거 (의존성 포함 고려)
- 검색 (`search`): 저장소 내 키워드 기반 패키지 탐색
- 정보 확인 (`info`): 버전, 아키텍처, 용량 등 상세 명세 확인
- 기록 관리 (`history`): 설치/삭제 작업 이력 조회 및 되돌리기(undo)

## 3. 저장소 (Repository) 설정

- 설정 경로: `/etc/yum.repos.d/` 디렉터리
- 파일 형식: `.repo` 확장자의 텍스트 파일
- 목록 확인: `dnf repolist` (활성화된 저장소 식별)
- 핵심 파라미터:
    - `baseurl`: 패키지 서버의 실제 주소
    - `enabled`: 저장소 활성화 여부 (1: 사용, 0: 미사용)
    - `gpgcheck`: GPG 서명을 통한 패키지 위변조 방지

## 4. 사용자 및 그룹 관리

- 사용자 생성 (`useradd`): 신규 계정 및 홈 디렉터리 생성
- 사용자 수정 (`usermod`): 보조 그룹 추가(`aG`), 쉘 변경 등 정보 수정
- 사용자 삭제 (`userdel -r`): 계정 및 홈 디렉터리 데이터 일괄 삭제
- 그룹 관리: `groupadd`(생성), `groupmod`(수정), `groupdel`(삭제)
- 암호 설정 (`passwd`): 사용자 비밀번호 부여 및 만료 관리

## 5. 권한 및 소유권 관리

- chmod (권한 변경): 읽기(4), 쓰기(2), 실행(1) 권한 부여
- chown (소유권 변경): `사용자:그룹` 형태의 소유권 이전
- chgrp (그룹 변경): 파일/디렉터리의 소속 그룹만 변경
- 권한 표기 방식:
    - 숫자 모드: 755(rwxr-xr-x), 644(rw-r--r--) 등 합산 수치 활용
    - 심볼릭 모드: `u+x`(실행 추가), `g-w`(쓰기 제거) 등 기호 활용
- 대상 구분: `u`(소유자), `g`(그룹), `o`(기타), `a`(전체)
