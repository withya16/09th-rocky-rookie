# RHEL 10 기초 정리

## 1. 핵심 개념 (RHEL 10 Official Highlights)

개요: 기업 IT 인프라에서 서버 운영체제 선택은 서버의 안정성, 보안, 장기적인 인프라 전략까지 영향을 미친다.  
과거에는 많은 기업들이 **CentOS**를 기본 서버 운영체제로 활용했지만, CentOS Stream 중심 구조로 전환되면서 새로운 리눅스 플랫폼을 선택해야 하는 상황이 되었다.

그래서 현재 기업 환경에서 가장 많이 검토되는 운영체제는 다음과 같다.

### Red Hat Enterprise Linux (RHEL)

대표적인 **상용 기업용 리눅스 플랫폼**

장점
- 안정적인 업데이트 정책
- 공식 기술 지원 제공
- 다양한 엔터프라이즈 솔루션과 높은 호환성
- 금융, 공공기관, 대기업에서 널리 사용

단점
- 라이선스 비용 발생

---

### Rocky Linux

CentOS 공동 창립자가 주도하여 개발한 리눅스 배포판으로  
**RHEL과 완전한 바이너리 호환성**을 목표로 만들어진 대표적인 대안 플랫폼이다.

장점
- RHEL 수준의 안정성 유지
- 무료 사용 가능
- 커뮤니티 기반 거버넌스 구조
- 특정 기업 정책 변화 영향이 적음
- 기존 CentOS 기반 서버 환경에서 거의 동일하게 동작하여 전환이 쉬움

---

### Ubuntu Server

Canonical이 개발한 리눅스 배포판으로 **클라우드와 개발 환경에서 널리 사용**된다.

장점
- 빠른 업데이트 주기
- 풍부한 패키지 생태계
- 많은 오픈소스 프로젝트가 Ubuntu 환경 기준으로 개발됨
- 개발환경 구축이 쉬움
- AWS, GCP 등 퍼블릭 클라우드 기본 이미지로 많이 제공
- Docker, Kubernetes 등 컨테이너 플랫폼과 높은 호환성

단점
- RHEL 계열 대비 기업용 상용 솔루션 호환성은 다소 낮은 편

# 2. 디렉터리 구조와 파일 탐색

리눅스는 **계층형 파일 시스템(Hierarchical File System)** 구조를 가진다.

## 주요 디렉터리

**FHS (Filesystem Hierarchy Standard)**  
RHEL 10도 표준 리눅스 계층 구조를 따른다.

| 디렉터리 | 역할 |
|---|---|
| / | 루트 디렉터리 |
| /home | 일반 사용자 홈 |
| /root | 관리자 홈 |
| /bin | 기본 명령어 |
| /etc | 시스템 설정 파일 |
| /usr | 사용자 프로그램 및 라이브러리 |
| /var | 로그 및 가변 데이터 |

# 3. 실습

## 3.1 디렉터리 구조

리눅스는 Windows처럼  
C 드라이브, D 드라이브 구조가 아니라 하나의 **트리 구조**를 사용한다.

주요 개념

- `/` (Root)  
  최상위 디렉터리. 모든 파일 시스템의 시작점

- `~` (Home)  
  현재 로그인한 사용자의 홈 디렉터리  
  예: `/home/username`

- `.` (Current Directory)  
  현재 위치

- `..` (Parent Directory)  
  현재 디렉터리의 상위 디렉터리

---

## 3.2 파일 탐색 및 이동

현재 위치 확인, 파일 목록 확인, 디렉터리 이동은 리눅스의 기본 작업이다.

| 명령어 | 의미 | 사용법 |
|---|---|---|
| pwd | Print Working Directory | 현재 경로 확인 |
| ls | List | 파일 목록 확인 (`-l`: 상세, `-a`: 숨김 포함) |
| cd | Change Directory | 디렉터리 이동 |

예시

```bash
pwd
ls -l
cd Documents
cd ..
