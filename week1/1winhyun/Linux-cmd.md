# 리눅스 디렉터리 구조와 파일 탐색 (FHS)

리눅스의 모든 파일과 디렉터리는 **루트(Root, `/`)**라는 최상위 디렉터리에서 시작하는 역트리 구조(Filesystem Hierarchy Standard, FHS)를 가집니다. 리눅스에서는 시스템 하드웨어부터 프로세스까지 모든 것을 파일로 취급합니다.

## 1. 주요 디렉터리 구조

| 디렉터리 | 설명 |
|----------|------|
| `/` (Root) | 파일 시스템의 최상위 디렉터리입니다. 일반 사용자는 이 디렉터리에 파일을 생성할 수 없습니다. |
| `/bin` & `/sbin` | `/bin`은 모든 사용자가 사용할 수 있는 필수 기본 명령어(예: `ls`, `cp`)를 포함하며, `/sbin`은 시스템 관리자가 사용하는 시스템 제어 명령어(예: `reboot`, `fdisk`)를 포함합니다. |
| `/boot` | 시스템 부팅에 필요한 커널과 부트로더(GRUB 등) 파일이 위치합니다. |
| `/dev` | 하드 드라이브, 터미널, USB 등 시스템에 연결된 하드웨어 장치 파일들이 위치합니다. |
| `/etc` | 시스템 전체 및 애플리케이션의 환경 설정 파일들이 저장되는 곳입니다. |
| `/home` | 일반 사용자들의 개인 디렉터리(홈 디렉터리)가 위치하며, 개인 파일과 설정을 저장합니다. |
| `/root` | 시스템 관리자인 최고 관리자(root) 전용 홈 디렉터리입니다. |
| `/var` | 로그 파일, 캐시, 스풀(spool) 등 시스템 운영 중 내용이 수시로 변경되는 가변 데이터가 저장됩니다. |
| `/usr` | 사용자 유틸리티, 애플리케이션, 공유 라이브러리 등 대부분의 소프트웨어가 설치되는 곳입니다. |
| `/tmp` | 프로세스 실행 중 생성되는 임시 파일이 저장되며, 시스템 재부팅 시 삭제됩니다. |
| `/proc` | 실행 중인 프로세스와 시스템 리소스(메모리, CPU 등)에 대한 정보를 텍스트 형태로 제공하는 가상 파일 시스템입니다. |

## 2. 경로(Path) 탐색의 개념

- **절대 경로 (Absolute Path)**: 항상 최상위 루트 디렉터리(`/`)부터 시작하여 대상 위치까지의 전체 경로를 의미합니다(예: `/home/user/file.txt`).
- **상대 경로 (Relative Path)**: 현재 작업 중인 디렉터리를 기준으로 대상의 위치를 나타냅니다.
  - `.` : 현재 디렉터리
  - `..` : 한 단계 위의 부모(상위) 디렉터리

# 파일 탐색 및 기본 명령어 실습

아래의 명령어와 옵션들을 조합하여 직접 터미널에서 실습해 보세요.

## ① `pwd` (Print Working Directory)

현재 위치한 작업 디렉터리의 경로를 화면에 출력합니다.

| 옵션 | 설명 |
|------|------|
| `-L` | 환경 변수나 심볼릭 링크(바로가기)를 포함한 논리적인 경로를 출력합니다. |
| `-P` | 심볼릭 링크를 해석하여 실제 원본이 위치한 물리적인 절대 경로를 출력합니다. |

## ② `cd` (Change Directory)

다른 디렉터리로 이동할 때 사용합니다.

| 사용법 | 설명 |
|--------|------|
| `cd <디렉터리 경로>` | 지정한 경로로 이동합니다. |
| `cd` 또는 `cd ~` | 현재 사용자의 홈 디렉터리로 즉시 이동합니다. |
| `cd ..` | 현재 위치에서 한 단계 위의 부모 디렉터리로 이동합니다. |
| `cd ../..` | 두 단계 위의 부모 디렉터리로 이동합니다. |
| `cd -` | 직전에 머물렀던 이전 디렉터리로 돌아갑니다. |

## ③ `ls` (List)

현재 디렉터리나 지정한 디렉터리의 파일과 폴더 목록을 출력합니다.

| 옵션 | 설명 |
|------|------|
| `-l` | 권한, 소유자, 크기, 수정 날짜 등 상세 정보(Long format)를 함께 출력합니다. |
| `-a` | `.`으로 시작하는 숨김 파일을 포함하여 모든 파일을 출력합니다. |
| `-R` (또는 `-r`) | 하위 디렉터리의 내용까지 모두 재귀적으로 나열합니다. |
| `-i` (또는 `-I`) | 각 파일의 고유 아이노드(inode) 번호를 표시합니다. |
| `-h` | 파일 크기를 사람이 읽기 쉬운 단위(KB, MB, GB)로 변환해 출력합니다 (`-l` 옵션과 함께 사용). |
| `-d` | 디렉터리 내부 내용이 아닌, 디렉터리 자체의 정보를 출력합니다. |
| `-s` | 할당된 파일 크기를 블록 단위로 표시합니다. |
| `-g` | `-l` 옵션과 같지만 파일 소유자 정보를 생략하고 출력합니다. |
| `-A` | `.`(현재)와 `..`(부모) 디렉터리 항목만 제외하고 모든 숨김 파일을 표시합니다. |
| `-F` | 디렉터리는 `/`, 실행 파일은 `*`, 심볼릭 링크는 `@` 기호를 붙여 파일 유형을 구분해 표시합니다. |
| `-X` | 파일 확장자를 기준으로 알파벳순으로 정렬하여 표시합니다. |
| `-C` | 결과를 열(Column) 형태로 정렬하여 출력합니다. |

## ④ `cp` (Copy)

파일이나 디렉터리를 복사합니다.

| 옵션 | 설명 |
|------|------|
| `-r` 또는 `-R` | 디렉터리를 복사할 때 하위 항목까지 모두 재귀적으로 복사합니다. |
| `-i` | 덮어쓸 대상 파일이 존재할 경우 확인(Interactive) 메시지를 띄웁니다. |
| `-f` | 확인하지 않고 강제로 덮어씁니다. |
| `-p` | 원본 파일의 소유자, 권한, 타임스탬프 속성을 그대로 유지하여 복사합니다. |
| `-L` | 심볼릭 링크를 복사할 때 링크가 아닌 원본 파일 자체를 복사합니다. |
| `-s` | 복사 대신 원본에 대한 심볼릭 링크 파일을 생성합니다. |
| `-v` | 복사 과정을 상세히 출력합니다 (보통 `-iv` 형태로 많이 쓰임). |

## ⑤ `mv` (Move)

파일이나 디렉터리의 위치를 이동하거나 이름을 변경할 때 사용합니다.

| 옵션 | 설명 |
|------|------|
| `-i` | 이동할 경로에 동일한 파일이 있으면 덮어쓸지 묻는 대화형 프롬프트를 띄웁니다. |
| `-f` | 파일이 존재하더라도 묻지 않고 강제로 덮어씁니다. |
| `-n` | 대상 위치에 동일한 파일이 있으면 덮어쓰지 않고 이동을 취소합니다. |
| `-v` | 파일이 이동되거나 이름이 변경되는 과정을 상세하게 출력합니다. |

## ⑥ `rm` (Remove)

파일이나 디렉터리를 삭제합니다. 리눅스는 휴지통 개념이 기본적으로 없으므로 주의해서 사용해야 합니다.

| 옵션 | 설명 |
|------|------|
| `-r` 또는 `-R` | 폴더와 그 안의 하위 디렉터리, 파일들을 모두 재귀적으로 삭제합니다. |
| `-i` | 삭제하기 전에 정말 지울 것인지 매번 확인을 요청합니다. |
| `-f` | 경고 메시지 없이 강제로 모든 파일을 삭제합니다. |

> **주의**: `rm -rf /` 같은 명령어는 시스템 전체를 파괴할 수 있어 매우 위험합니다.

# 도움말 활용 방법 (`man`, `--help`)

리눅스는 명령어의 옵션을 모두 외우지 않아도 스스로 학습하고 문제를 해결할 수 있도록 내장 도움말을 제공합니다.

## ① `--help` 옵션

명령어 뒤에 붙여 사용하며, 오프라인 환경에서 명령어의 목적과 사용 가능한 매개변수, 옵션을 간략하게 보여줍니다.

```bash
ls --help
```

## ② `man` (Manual Pages)

명령어, 시스템 호출, 설정 파일 등에 대한 매우 상세한 공식 매뉴얼을 제공합니다. 매뉴얼은 주로 이름(Name), 개요(Synopsis), 설명(Description), 옵션(Options), 관련 항목(See Also), 작성자(Author) 등의 구조로 나뉘어 있습니다.

### 기본 사용법

```bash
man <명령어>    # 해당 명령어의 매뉴얼을 확인합니다.
```

### 주요 옵션

| 옵션 | 설명 |
|------|------|
| `-k <키워드>` | 특정 키워드가 포함된 모든 매뉴얼 페이지를 검색하여 나열합니다. |
| `-f <명령어>` | 명령어의 간단한 설명과 어느 섹션에 포함되어 있는지 표시합니다. |
| `-a <명령어>` | 이름이 같은 여러 매뉴얼 페이지를 순차적으로 모두 보여줍니다. |
| `-K` (또는 `-wK`) | 모든 매뉴얼 파일의 본문까지 뒤져서 특정 단어를 포함하는 매뉴얼을 찾습니다. |

### 특정 섹션 검색

리눅스 매뉴얼은 1번(사용자 명령어), 2번(시스템 호출), 3번(라이브러리 함수), 5번(파일 포맷), 8번(시스템 관리 명령어) 등으로 나뉩니다.

```bash
man 3 sleep    # sleep 명령어 대신 C 라이브러리 함수 sleep의 매뉴얼을 확인
```

### 매뉴얼 창 내부 단축키 (less 뷰어 기반)

| 단축키 | 설명 |
|--------|------|
| 방향키(`Up`/`Down`) 또는 `Enter` | 한 줄씩 위아래로 이동합니다. |
| `Space bar`, `PgDn`, `D` | 한 페이지 아래로 이동합니다. |
| `PgUp`, `B` | 한 페이지 위로 이동합니다. |
| `/검색어` | 매뉴얼 내에서 해당 문자를 아래 방향으로 검색합니다 (예: `/option`). |
| `?검색어` | 위쪽 방향으로 문자열을 검색합니다. |
| `n` | 검색 결과의 다음 일치 항목으로 이동합니다. |
| `N` (또는 `Shift + N`) | 검색 결과의 이전 일치 항목으로 이동합니다. |
| `h` | less 뷰어 자체의 도움말(단축키 요약)을 봅니다. |
| `q` | 매뉴얼 읽기를 종료하고 터미널로 복귀합니다. |
