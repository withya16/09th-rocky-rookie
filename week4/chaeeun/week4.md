## Systemd

- 커널이 올라온 뒤 사용자 공간에서 시스템 전체를 시작하고, 서비스의 생명주기를 관리하는 관리자
- PID 1
- 모든 프로세스 트리의 최상위
    - 실패한 서비스 재시작
    - 종료 코드 해석
- Unit : systemd의 관리 대상 단위
    - 사용가능한 systemd 장치 유형
    
    | **단위 유형** | **파일 확장자** | **설명** |
    | --- | --- | --- |
    | 서비스 단위 | `.service` | 시스템 서비스 |
    | 대상 단위 | `.target` | systemd 장치 그룹 |
    | 자동 마운트 단위 | `.automount` | 파일 시스템 자동 마운트 지점 |
    | 장치 단위 | `.device` | 커널에서 인식한 장치 파일 |
    | 마운트 단위 | `.Mount` | 파일 시스템 마운트 지점 |
    | 경로 단위 | `.path` | 파일 또는 디렉터리 |
    | 범위 단위 | `.scope` | 외부에서 생성된 프로세스 |
    | 슬라이스 단위 | `.slice` | 시스템 프로세스를 관리하는 계층적으로 구성된 단위 그룹 |
    | 스냅샷 단위 | `.snapshot` | 저장된 systemd 관리자 상태 |
    | 소켓 단위 | `.socket` | 프로세스 간 통신 소켓 |
    | 스왑 장치 | `.swap` | 스왑 장치 또는 스왑 파일 |
    | 타이머 장치 | `.timer` | systemd 타이머 |
- 의존성
    
    
    | 옵션 | 의미 |
    | --- | --- |
    | `After=` | 이 unit보다 뒤에 시작 |
    | `Before=` | 이 unit보다 먼저 시작 |
    | `Requires=` | 강한 의존성, 실패하면 해당 프로세스도 실패 |
    | `Wants=` | 약한 의존성, 있으면 좋지만 필수는 아님 |

### systemctl 명령어

- status: 현재 상태 확인
- start: 현제 세션에서 서비스 시작
- stop: 현재 떠있는 서비스 중지
- restart: 중지 후 다시 시작
- reload: 프로세스를 죽이지 않고 설정 재적용
- enable: 부팅 시 자동 시작되도록 연결 생성
- disable: 부팅 자동 시작 해제

## 프로세스

- 프로세스: 메모리에 적재되어 실행 중인 상태의 프로그램
- PID: 프로세스 고유번호
- PPID: 부모 프로세스 번호
- Foreground: 현재 터미널에서 직접 제어중인 작업
- Background: 뒤에서 실행되는 작업
- NICE 값 : CPU 스케줄링 우선순위
    - -20 ~ 19
    - 작을 수록 우선순위가 높음

### 프로세스 조회

```bash
ps
```

- -a : 다른 사용자의 프로세스 포함
- -u : 사용자 중심 형식으로 출력
- -x : 터미널에 연결되지 않은 프로세스 포함

### 프로세스 트리 구조 보기

```bash
pstree
```

- 부모-자식 관계를 트리로 표시
- -p : PID 포함

### 실시간 조회

```bash
top
```

- CPU, 메모리 사용량을 보여줌
- top 안에서 사용하는 키
    - P : CPU 사용량 기준 정렬
    - M : 메모리 사용량 기준 정렬
    - k : 프로세스 종료
    - r : nice 값 변경
    - q : 종료

### 프로세스 상태

- R : 실행 중 또는 실행 대기
- S : interruptible sleep, 일반 대기
- D : uninterruptible sleep, 디스트 I/O 대기
- T : 정지됨
- Z : 좀비 상태
- I : idle 커널 스레드

### 프로세스 종료

```bash
kill PID
```

- 정상 종료 요청
- -9 : 강제 종료

## 작업 스케줄링

- 정해진 시간이나 조건에 따라 자동실행 되도록 스케줄링
- cron : 주기적 실행
- at : 일회성 실행

### cron

- 정해진 주기에 맞춰 명령이나 스크립트를 실행하는 데몬
- crond : 실제 스케줄 실행하는 데몬
- crontab : 사용자별 스케줄 테이블

### crontab 설정

```bash
corntab [-option]
```

- -e : 현재 사용자 스케줄 편집
- -l : 현재 사용자 스케줄 조회
- -r : 현재 사용자 스케줄 삭제

```bash
분 시 일 월 요일 명령어
```

- 빈칸/모든값은 *로 표시
    - 0 2 * * * /backup.sh : 매일 새벽 2시에 실행
    - * /5 * * * * /ckeck.sh : 5분마다 실행

### at

- 특정 시간에 한번만 실행

```bash
at [특정시간]
> 실행할 명령어 입력
Ctrl+D (종료)
```

- 특정시간은 now + 10 minutes / 9:00 tomorrow 같은 형식 가능

### 예약 목록 확인

```bash
atq 
```

### 예약 삭제

```bash
atrm [작업번호]
```

## cgroup

- 프로세스 그룹 단위 자원 제어
- 여러 프로세스를 하나의 그룹으로 묶고, 해당 그룹에 자원 사용 정책을 적용
- slice : systemd가 cgroup을 논리적으로 나누기 위해 사용하는 개념

### 자원제한

```bash
# CPU 제한
systemctl set-property nginx.service CPUQuota=50%

# 메모리 제한
systemctl set-property nginx.service MemoryLimit=500M
```
